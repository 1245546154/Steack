Raid 1988年（廉价的冗余磁盘阵列）
容量，性能，可靠性。

镜像和数据校验技术，条带。 

raid分为不同的等级
RAID0 - RAID6 

业界公认的为RAID0 - RAID5 
使用最多的是0.1.3.5.6.10

公司的raid 都是硬盘级别的，RAID5使用的很多。

软RAID：所有功能均有操作系统和CPU来完成，没有RAID控制，（需要消耗CPU的资源）
硬RAID：配备了专门的RAID控制 处理芯片 和I/O处理芯片 以及阵列缓冲，不占用CPU资源，（但是成本高）
软硬RAID：介于软硬中间

RAID 是一类多磁盘管理技术，其向主机环境提供了成本适中，数据可靠性高的高性能存储。

RAID的两个关键目标是提高数据可靠性和I/O（输入输出）性能

条带化，当多个进程同时访问一个磁盘时，可能会出现磁盘冲突。
条带化技术就是一种自动的将 I/O 的负载均衡到多个物理磁盘上的技术，将数据分块进行传输存放。

----------------------------------------------------------------------------------------------------------------------------

条带：是把连续的数据分割成相同大小的数据块，把每段数据分别写入到阵列中的不同磁盘上的方法。简单的说，条带是一种将多个磁盘驱动器合并为一个卷的方法。 许多情况下，这是通过硬件控制器来完成的。

创造一个阵列，将数据分散存储在所有的磁盘上。
缺点：没有数据校验，没有冗余性，磁盘坏了一块，就全坏了
优点：简单，明确。。速度快，空间利用性最大

----------------------------------------------------------------------------------------------------------------------------
RAID0    是一种简单的，无数据校验的数据条带化技术。实际上不是一种真正的RAID，因为（他不提供任何形式的冗余策略。）（别名：条带）

RAID1：  只能有2两块硬盘，互为镜像。（别名：镜像）

RAID5：  只能有3块硬盘，不存在RAID4 并发写操作时的校验盘性能瓶颈，另外RAID5，还具备很好的扩展性。

RAID5： 可以自动修复数据（非常智能化）――带有分散校验的数据条带 （校验一次）
左对称，默认为左旋校验，优先使用最后面一个盘做校验――left-symmetric

RAID6： 需要4块硬盘RAID6技术是在RAID 5基础上，为了进一步加强数据保护而设计的一种RAID方式（校验两次）

----------------------------------------------------------------------------------------------------------------------------

RAID组合等级 
RAID00：多RAID 0阵列组合形成一个RAID 00阵列，提供更佳的性能及容量大得多的驱动器。阵列最多可具有15个驱动器，用户可增加300%的阵列容量。

RAID01： 组合使用 当一个数据盘损坏时，系统根据同一条带的其他数据块和对应的校验数据来重建损坏的数据，与其他的RAID等级一样，重建数据时，RAID5的性能会受到较大的影响。先作镜像在做条带化，是对虚拟硬盘实现镜像。通常下，01比10具备更好的容错能力。

RAID10：它是利用奇偶校验实现条带集镜像，所以它继承了Raid0的快速和Raid1的安全,即把数据分割，而这分出来的每一路则再分两路，做Mirroring操作，即互做镜像。

安装软件：
yum install mdadm
rpm -qa | grep mdadm  


1.把硬盘创建分区	fdisk /dev/sdb
2.创建raid0	        mdadm -C /dev/md0 --l 0 -n 2 /dev/sdb1 /dev/sdc1   # -x 是备盘
3.查看raid状态	        cat /proc/mdstat
4.查看某阵列	        mdadm -D /dev/md0     #-a添加一个硬盘进去
5.创建挂载点	        mkdir /md0
6.格式化raid分区	mkfs.xfs /dev/md0
7.挂载		        mount -t xfs /dev/md0 /md0
9.查看挂载              df -h


mdadm /dev/md1 --fail /dev/sdh --remove /dev/sdh  #删除创建的md分区

----------------------------------------------------------------------------------------------------------------------------

创建raid5
mdadm -C /dev/md1 --l 5 -n 3 -x 1 /dev/sd{g,h,i,j}    #创建完md1之后会自动同步0和1 做镜像
cat /proc/mdstat
mkdir /md5
mkfs.xfs /dev/md5
mount -t xfs /dev/md1 /md5
df -h


----------------------------------------------------------------------------------------------------------------------------

创建raid6
mdadm -C /dev/md1 -a yes -l 6 -n 4 /dev/sdg /dev/sd{m,l,k,g}    #创建完md1之后会自动同步0和1 做镜像
cat /proc/mdstat
mdadm -D /dev/md6     
mkdir /md6
mkfs.xfs /dev/md6
mount -t xfs /dev/md1 /md6
df -h

----------------------------------------------------------------------------------------------------------------------------

创建raid1  
mdadm -C /dev/md1 -l 1 -n 2  /dev/sd{c,d}   #创建完md1之后会自动同步0和1 做镜像
cat /proc/mdstat
mdadm -D /dev/md1     
mkdir /md1
mkfs.xfs /dev/md1
mount -t xfs /dev/md1 /md1
df -h
